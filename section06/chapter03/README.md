
# 기능 구현하기
## 🧠 State Lifting (상태 끌어올리기)

### ❓ 어디에 state(여기선 counter) 를 만들어야 할까?

현재 카운터의 값을 저장하는 `useState`를  
👉 **App.jsx / Controller.jsx / Viewer.jsx** 셋 중 어디다 관리할까?

👉 **정답: App.jsx**

---

### 💡 이유

- `Viewer`와 `Controller`는 **부모-자식 관계가 아님**
- 둘은 같은 부모를 가진 **형제 관계**

👉 따라서 서로 직접 데이터를 주고받을 수 없음

---

### 🔥 React에서 데이터 전달 방법

👉 컴포넌트 간 데이터 전달은 **props**를 사용

❗ 근데 중요한 특징:

- props는 **오직 부모 → 자식 방향으로만 전달 가능**

---

### 🚨 그래서 결론

👉 형제 컴포넌트끼리는 직접 데이터 공유 불가능  
👉 **공통 부모에서 state를 관리해야 함**

---

### 🎯 해결 방법

👉 **App.jsx (공통 부모)에 state를 만든다**

```txt id="tree01"
App (state 보유)
 ├── Viewer (값 받기)
 └── Controller (값 변경)
````

---

### 📌 이 개념을 뭐라고 하냐?

👉 **State Lifting (상태 끌어올리기)**

* 여러 컴포넌트가 공유해야 하는 상태를
* **가장 가까운 공통 부모로 올리는 것**

---
